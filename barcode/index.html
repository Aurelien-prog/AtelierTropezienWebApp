ActiveBarcode for PHP

You can use ActiveBarcode (activebarcode.ocx) to create barcodes with PHP (on IIS, PWS or Apache): The barcodes are created on server side and no modifications (e.g. plug in's) are required for the client side. Every standard Internet browser that supports PNG Images will show the barcodes.

We have included a PHP file (barcodeimage.PHP) that works as a wrapper around the activebarcode.ocx*. So it's very easy for you to create dynamic barcodes. Simply use the barcodeimage.php inside the SRC-Attribute of an IMG tag. The PHP file will return a barcode image to the client. The sourcecode of the barcodeimage.php file can be used everywhere and you are allowed to make modifications to it.

Here are some examples for IMG-Tags:
1. This is the simplest example that is possible. It has no parameters, only the default values are used.
<img src="barcodeimage.php" />



2. This example shows how to set the most important parameters allowing us to get a useful barcode.
<img src="barcodeimage.php?code=CODE128&text=ActiveBarcode&showtext=1&width=360&height=100&borderwidth=0" width="360" height="100" />



Note: It's required to set the width and height parameters as URI-Args for barcodeimage.php as shown here (otherwise the barcode will have default dimensions that may not fit your requirements). And it's a good idea (not required) to set the parameters also to the IMG Tag. This way the browser will know how much space the resulting image will take on the page, even if it has not loaded yet.
3. This example has it all: custom size, color, rotation and borders around the barcode.
<img src="barcodeimage.php?code=CODE39EXTENDED&text=Hello%20World!&showtext=1&rotate=90&backcolor=002288&forecolor=12345&width=250&height=430&borderwidth=50&borderheight=25" width="250" height="430" />



Note: URI-Args are URL-Encoded (i.e. a space is %20).
Backcolor and forecolor can be set as html color (ffffff = white, 000000 = black).
4. This example shows the 2D barcode PDF417 with the encoded text "Hello World!"
<img src="barcodeimage.php?code=PDF417&text=Hello%20World!&showtext=1&width=360&height=100&borderwidth=0" width="360" height="100" />



5. This example shows a QR Code, encoded with a control character. "Active" is encoded and followed by a Carriage Return and the word "Barcode"
<img src="barcodeimage.php?typeno=113&text=Active<CR>Barcode&showtext=1&width=150&height=150&borderwidth=0" />



More detailed info about the barcodeimage.php and the parameters that can be passed can be found in the documented sourcecode of the barcodeimage.php file.

These examples just show the basic usage of IMG-Tags. If you want to create barcodes dynamically you can use php, Perl, asp etc. in order to create IMG-Tags with dynamic parameters. We have placed some php-example-Files into the /php folder of you installation:
  • example_create_current_time_as_code128.php - Shows the servertime as a barcode.
  • example_calc_checksum_only.php - Show the usage of the component without creating barcodes as an image stream - i.e. to calculate barcode-checksums.
ActiveBarcode is written in C++ . We have stresstested it under IIS and optimzed it for stability and speed so that you can use this component even on high traffic servers within the adress-space of IIS.
IMPORTANT NOTE: If you see broken pictures instead of barcodes here, activebarcode.ocx is not yet installed for use under IIS.
